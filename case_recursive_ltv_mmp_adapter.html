MMP Schema Adapter

src/ingestion/mmp_client.py now supports multiple response formats and maps them into the normalized cohort schema.

Supported payload shapes

  1. Flat rows
[{"game_name":"NS","platform":"iOS","install_date":"2026-01-10","actual_ltv_d0":1.0}]
  1. Wrapped rows
{"data":[{"game_name":"NS","install_date":"2026-01-10","actual_ltv_d0":1.0}]}
  1. Nested dimensions/metrics per row
{"rows":[{"dimensions":{"game_name":"NS","install_date":"2026-01-10"},"metrics":{"cost":100,"actual_ltv_d0":1.0}}]}
  1. Array rows + columns
{"columns":["game_name","platform","network_name","campaign_name","install_date","currency","cost","actual_ltv_d0"],"rows":[["NS","iOS","TikTok","C1","2026-01-10","USD",100,1.0]]}

  • LTV horizon prefixes
  • MMP_LTV_METRIC_PREFIXES
  • ROAS horizon prefixes
  • MMP_ROAS_METRIC_PREFIXES

Default LTV prefixes are actual_ltv_d,lifetime_value_d,revenue_d,ltv_d, and default ROAS prefixes are actual_roas_d,roas_d, so keys like lifetime_value_d7 and roas_d7 are parsed automatically.

Notes

  • Cohort timezone is UTC.
  • Revenue basis is total revenue (IAP + ad).
  • RS API endpoint is GET https://automate.mmp.com/reports-service/report.
  • Required RS-style params include date_period and app_token__in.
  • If payload shape is unknown, ingestion throws an explicit format error.
  • MMP_API_KEY and MMP_APP_TOKEN can come from .env or the dashboard credentials vault.
  • For strict bucket install policy, include both metrics in pulls:
  • network_installs (used for non_incent)
  • installs (used for incent)
  • For strict bucket cost policy, include both metrics in pulls:
  • network_cost (used for non_incent)
  • mmp_cost (used for incent)
