LTV / ROAS Forecasting Engine — Architecture

Two-layer prediction model with post-processing pipeline and geo-aware residual calibration
INDUSTRY PROJECT · ARCHITECTURE

Architecture

Analysis date: 2026-02-27

Overview

The engine uses a two-layer design. The first layer builds reference curves — baseline LTV growth patterns fitted to historical cohort data, segmented by OS, geo bracket, and channel. The second layer applies campaign-level scaling factors on top of those reference curves. Most of the LTV curve shape is captured at the segment level; campaign-level variation shows up as a scalar multiplier over the reference.

Why the layers are separate

The reference curve builder and the prediction engine are separate modules deliberately. Backtesting requires replaying the model at historical cutoffs — simulating what the prediction would have been if you'd only had D7 data at the time. If the two were coupled, a clean simulation would be impossible without contaminating the reference curves with future data. Keeping them separate means live predictions and backtest simulations go through the same code path, just with different data windows fed in.

Data flow

Training: raw cohort data arrives in wide format (one row per cohort, one column per day), gets reshaped to long format, then the reference curve builder fits power-law curves per segment. The fallback hierarchy — Country → Geo Bracket → OS → Global — handles sparse segments gracefully. If there aren't enough cohorts for a reliable country-level fit, the builder falls back to the geo bracket level, and so on.

Prediction: given a current cohort with data up to input day D, the engine looks up the reference curve for that cohort's OS/geo/channel segment, applies the campaign scaling factor, and generates forecasts from D1 through D390. Tail factors handle D120+ extrapolation separately — the long tail behaves differently from the early growth window and warrants its own fitting step.

Backtest: for each historical input day in a defined set (D1, D3, D7, D14, D30, ...), the engine simulates a prediction using only data available at that cutoff, then compares against the actuals that materialized afterward. Output is an accuracy matrix across horizons and input days, exported to Excel.

Error handling

The model degrades gracefully rather than failing hard. Curve fitting uses R² thresholds — below the threshold it falls back one level in the segment hierarchy. Campaign scaling factors are clipped to defined bounds to prevent a sparse campaign from producing a nonsensical multiplier. The MMP API client retries on 429 and 5xx with exponential backoff.
