Architecture

Pattern Overview

Overall: Two-layer machine learning prediction system with data pipeline architecture

Key Characteristics: - Layer 1: Log-linear reference curves at OS × Geo Bracket × Channel level - Layer 2: Weighted-average scaling factors at Campaign level - Backtest-driven validation with accuracy matrix generation - Command-line driven batch processing system

Layers

Data Ingestion Layer: - Purpose: Fetch and preprocess external marketing data - Location: mmp_data_load.py - Contains: API client, data transformation, CSV export - Depends on: MMP API, pandas, requests - Used by: Main prediction engine

Reference Model Layer: - Purpose: Build baseline prediction curves from historical data - Location: ltv_roas_engine_v3.21.py (functions: build_reference_curves, build_retention_reference, build_tail_reference) - Contains: Curve fitting algorithms, statistical models, fallback mechanisms - Depends on: scipy.optimize, numpy mathematical functions - Used by: Prediction generation layer

Prediction Generation Layer: - Purpose: Generate LTV/ROAS forecasts using reference models and scaling factors - Location: ltv_roas_engine_v3.21.py (functions: generate_predictions, get_ref_pred_curve) - Contains: Multi-horizon prediction logic, campaign-level adjustments - Depends on: Reference model layer, retention calculations - Used by: Output and validation layers

Backtest Validation Layer: - Purpose: Validate prediction accuracy across historical time windows - Location: ltv_roas_engine_v3.21.py (functions: run_backtest, generate_backtest_summary) - Contains: Time-series cross-validation, accuracy metrics, Excel report generation - Depends on: Prediction generation layer - Used by: Model evaluation and tuning

Data Flow

Training Flow:

  1. Load historical cohort data via load_adjust_data()
  2. Reshape from wide to long format via reshape_long()
  3. Build reference curves using build_reference_curves()
  4. Calculate retention references via build_retention_reference()
  5. Generate tail factors using build_tail_reference()

Prediction Flow:

  1. Load current cohort data
  2. Lookup reference curves by OS/Geo/Channel hierarchy
  3. Apply campaign-specific scaling factors
  4. Generate predictions for D1-D390 horizons
  5. Export to CSV with reference curve metadata

Backtest Flow:

  1. For each historical input day (D1, D3, D7, D14, etc.)
  2. Simulate prediction using only data available at that day
  3. Compare against actual observed LTV values
  4. Generate accuracy matrices and Excel reports

State Management: - Stateless functional design with immutable data transformations - Reference models built once and cached in memory - No persistent state between runs

Key Abstractions

Reference Curves: - Purpose: Baseline LTV growth patterns by segment - Examples: ltv_roas_engine_v3.21.py lines 629-836 - Pattern: Power law curve fitting with fallback hierarchy (Country → Geo Bracket → OS → Global)

Scaling Factors: - Purpose: Campaign-level adjustments to reference curves
- Examples: ltv_roas_engine_v3.21.py lines 1538-1547 - Pattern: Weighted ratio calculation with clipping bounds

Tail Factors: - Purpose: Long-term (D120+) LTV extrapolation ratios - Examples: ltv_roas_engine_v3.21.py lines 911-1064 - Pattern: Install-weighted LTV(horizon)/LTV(D120) with vintage awareness

Bridge Corrections: - Purpose: Early-window shape corrections for specific channels - Examples: ltv_roas_engine_v3.21.py lines 1065-1187 - Pattern: Median ratio adjustments for input days ≤ D7

Entry Points

Main Prediction Engine: - Location: ltv_roas_engine_v3.21.py main() function (line 2160) - Triggers: Command-line execution with argparse - Responsibilities: Orchestrate full pipeline, handle CLI arguments, coordinate backtest and prediction modes

Data Collection Script: - Location: mmp_data_load.py main() function (line 128) - Triggers: Manual execution for data refresh - Responsibilities: Fetch marketing data from MMP API, preprocess and export to CSV

Error Handling

Strategy: Graceful degradation with fallback hierarchy

Patterns: - Reference curve fitting: Falls back from Country → Geo Bracket → OS → Global levels - API failures: Retry with exponential backoff, fail after max attempts - Sparse data: Use weighted medians and install-based shrinkage - Model validation: R² thresholds trigger fallback to simpler models

Cross-Cutting Concerns

Logging: Python logging module with INFO/WARNING levels for pipeline progress Validation: R² thresholds, monotonicity guards, SF clipping bounds Authentication: Bearer token authentication for MMP API
